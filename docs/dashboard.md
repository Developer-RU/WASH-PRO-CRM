React-приложение с тёмной/светлой темой, фиксированным боковым меню и **live-обновлением** данных без перезагрузки страницы.

## Навигация

| Группа | Разделы |
|--------|---------|
| **Главное** | Обзор |
| **Объекты** | Автомойки, Посты, Текущее состояние |
| **Карты** | Скидочные, Сервисные, VIP |
| **Аналитика** | Статистика использования, Финансовая статистика, Архивирование |
| **Система** | Уведомления, Резервные копии, Telegram, Валюты, Типы скидок, Логи |

Внизу сайдбара — блок **Resources** (документация и репозиторий на GitHub). Сайдбар закреплён слева; прокручивается только область контента.

## Live-режим

Оперативные данные обновляются автоматически (`usePolling`). В шапке — индикатор live-режима.

| Интервал | Страницы |
|----------|----------|
| 3 с | Текущее состояние постов |
| 5 с | Обзор |
| 10 с | Посты, карты, финансы, использование, уведомления |
| 15 с | Автомойки, архив, бэкапы, валюты, типы скидок, логи |

На странице **Текущее состояние** время режима тикает в реальном времени (`LiveModeTimer`).

## Таблицы (DataTable)

Все основные таблицы поддерживают:

- поиск по текстовым полям;
- сортировку по клику на заголовок;
- фильтры (выпадающие списки);
- пагинацию;
- массовые действия (экспорт CSV, удаление, смена статуса — где доступно).

## Разделы подробно

### Обзор (`/`)

KPI-карточки до инкассации: наличная / безналичная / общая выручка / сумма скидок / активные ошибки.

Суммы по финансам и использованию считаются как **сумма последних записей каждого поста** (не всех исторических строк).

Графики: статус постов (онлайн / офлайн / обслуживание / ошибка), круговая диаграмма использования по категориям, линейный график выручки. Таблица последних уведомлений.

### Автомойки (`/washes`)

Справочник объектов: название, описание, адрес, дата создания, облако. Колонка «Количество постов».

### Посты (`/posts`)

Посты с привязкой к автомойке: номер, название, серийный номер контроллера, объект. Каскадное удаление связанных данных (состояния, карты, статистика).

### Текущее состояние (`/states`)

Все посты всех объектов. Колонки:

| Колонка | Описание |
|---------|----------|
| Адрес объекта | Адрес автомойки |
| Текущий баланс | Баланс активной карты / из SCADA |
| Бесплатная пауза | Секунды |
| Сумма скидки | В валюте системы |
| Время режима | Live-таймер с момента включения режима |
| Режим | Название текущего режима |
| Дата и время | Последнее сообщение |

Если пост ещё не передавал данные — «Ожидание первых данных».

### Карты (`/cards`)

Три подраздела:

| Подраздел | `cardType` | Поля |
|-----------|------------|------|
| Скидочные | `regular` | Номер карты, тип скидки, баланс, сумма скидки, пост, статус, дата и время |
| Сервисные | `service` | Номер, начало/окончание действия, пост, статус, дата и время |
| VIP | `unlimited` | Номер, начало/окончание действия, пост, статус, дата и время |

**Тип скидки** — номер `1`–`5` из справочника; в интерфейсе отображается название (см. «Типы скидок»).

Статусы карт: `success` (Успешно), `rejected` (Отклонено).

### Статистика использования (`/usage`)

Две группы: **до инкассации** и **после инкассации**.

Карточки сверху: использование клиентами / сервисное / VIP — сумма **последних значений по каждому посту и категории**.

Таблица: пост (номер), адрес объекта, категория, время, запуски, клиенты, дата и время.

### Финансовая статистика (`/finance`)

Две группы: до / после инкассации.

Карточки: наличные, внешние (безнал), скидочные средства — сумма последних записей по каждому посту.

Таблица: пост, адрес объекта, наличные, безнал, скидочные, выручка, дата и время. Суммы в валюте из настроек.

### Архивирование (`/archive`)

Политики хранения по группам данных (карты, состояния постов, статистика использования, финансы). Журнал операций архивирования.

### Валюты (`/currency`, Admin)

Справочник валют (`/api/crm/currencies`): код ISO, название, символ, валюта по умолчанию для отображения в Dashboard.

### Типы скидок (`/discount-types`, Admin)

Справочник номеров **1–5** и их названий (`/api/crm/discount-types`). При старте `init-seed` создаёт значения по умолчанию. Названия редактируются в интерфейсе.

### Прочие разделы (Admin)

| Раздел | Описание |
|--------|----------|
| Резервные копии | Список, ручной запуск, удаление |
| Telegram | Токен бота, ID администраторов |
| Логи | Аудит Dynamic API с фильтрами |

## Роли пользователей

| Роль | Права в Dashboard |
|------|-------------------|
| **Administrator** | Полный доступ, включая бэкапы, Telegram, валюты, типы скидок, логи |
| **Operator** | Просмотр, создание, редактирование |
| **Viewer** | Только просмотр |

Роли настраиваются в Dynamic API Panel через группы пользователей.

## Демо-данные

```bash
# Автомойки, посты, статистика
./scripts/generate-demo-data.sh

# Карты (привязка к постам, типы скидок 1–5)
./scripts/generate-demo-cards.sh
```

## Сборка и разработка

```bash
cd dashboard
npm install
npm run dev      # http://localhost:5173
npm run build    # production в dist/
```

В Docker Dashboard проксирует `/api` на `dynamic-api:3001`.

## Технологии

- React 18 + TypeScript + Vite
- React Router, Tailwind CSS, Lucide Icons
- Recharts (графики на обзоре)
- Автообновление: `usePolling`, `LiveModeContext`
